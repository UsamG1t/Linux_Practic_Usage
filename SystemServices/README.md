## Самостоятельная работа

![](Attached_materials/P2P_systemd.png)

В качестве самостоятельной работы слушателям предлагается создать две ВМ и объединить их в сеть согласно топологии с помощью `systemd-networkd`. На ВМ необходимо описать и запустить четыре сетевых сервиса: два внутренних и два основных.
 + Сервис на PC1 — «Шифровщик»: получает на вход данные, и выдаёт на выход шестнадцатиричный вывод содержимого _в любом удобном для решения задачи_ формате
	 + Использовать для описания вывода лучше `xxd`, а не `hexdump`, у него есть нужные флаги
	 + Да, нужно будет какие-то из них указывать ☺
 + Сервис на PC1 — Сервер с «шифровщиком»: При подключении вызывает выполнение такого shell-сценария:

```
read args
./multitask.sh `echo $args` | netcat localhost <service1_port>
```

	 + Сценарий вызывает `multitask` и его вывод перенаправляет в «шифровщик», после чего вернёт их обратно
	 + Да, это тот самый multitask из задания про командную строку
	 + Укладывание в другой сценарий нужно для корректной работы сервиса

 + Сервис на PC2 — «Дешифровщик»: получает на вход шестнадцатиричные данные, и выдаёт на выход обычный вывод содержимого _в любом удобном для решения задачи_ формате
	 + Вот тут-то точно нужны будут флаги `xxd`
 + Сервис на PC2 — Клиент с «дешифровщиком»: При подключении вызывает выполнение такого shell-сценария:

```
netcat 10.0.10.1 <service2_port> | netcat localhost <service3_port>
```

	 + Сценарий обращается к серверу и его вывод перенаправляет в «дешифровщик», после чего выводит на экран ответ

Для отчёта воспользуйтесь форматом записи и получения отчётов из [лабораторных работ](https://github.com/UsamG1t/Nets_ASVK_Labs/blob/master/02_SystemGreetings/%D0%97%D0%BD%D0%B0%D0%BA%D0%BE%D0%BC%D1%81%D1%82%D0%B2%D0%BE%20%D1%81%20%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%BE%D0%B9.md#%D1%81%D0%B4%D0%B0%D1%87%D0%B0-%D1%81%D0%B0%D0%BC%D0%BE%D1%81%D1%82%D0%BE%D1%8F%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D1%8B%D1%85-%D1%80%D0%B0%D0%B1%D0%BE%D1%82). **Важно**: отчёты запускайте после всей настройки сети и сервисов. 

В отчётах выполните следующие команды:

 + `ss -ltp`
 + `cat /etc/systemd/network/<имя ваших конфигурационных файлов>`
 + `cat /etc/systemd/system/<имя ваших сервисов и сокетов к ним>`
 + `cat <все дополнительно описываемые файлы, например, маленькие побочные shell-сценарии>`
 + `netcat 10.0.10.2 <service4_port>` с любой командой внутри

Полученные отчёты `report.01.pc1` и `report.01.pc2` пришлите на почту преподавателя с указанием ФИО. Тема письма должна быть «\[LinuxPractice5\]<Фамилия> <Имя>»
