# Что такое процесс?

 + Техническое определение: _Процесс_  — совокупность машинных команд и данных, которая обрабатывается в рамках вычислительной системы и обладает правами на владение некоторым набором ресурсов
 + Суть:
	 + Программа — просто _текст_. Её исполняемый экземпляр со своими метаданными — _процесс_.
 + Много подробностей есть в [специальном материале](https://github.com/UsamG1t/jaffar_torrent/blob/master/Conspectology/00.C_Anthology/1.%20%D0%90%D0%BD%D1%82%D0%BE%D0%BB%D0%BE%D0%B3%D0%B8%D1%8F%20%D1%8F%D0%B7%D1%8B%D0%BA%D0%B0%20C%20(%D0%9C%D0%BD%D0%BE%D0%B3%D0%BE%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D1%81%D1%81%D0%BD%D0%BE%D0%B5%20%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5).md) для второго курса ВМК

## Состав процесса:

 + `PID`, `PPID` — идентификаторы процесса и родителя
 + `UID / EUID / GID / EGID` — Идентификаторы пользователя
 + Аргументы командной строки
 + Окружение процесса
 + Виртуальное адресное пространство
 + Состояние процесса
 + И так далее…

```console
$ sleep 300 &
[1] 12345

$ ps -o pid,ppid,pgid,tty,stat,user,comm,args -p 12345
  PID  PPID  PGID   TT       STAT USER     COMMAND         COMMAND
12345  6789 12345  tty1    S    user     sleep           sleep 300

$ cat /proc/12345/status
$ ls -l /proc/12345/fd
```

## Жизненный цикл

 + _fork_ — Создание сыновьего процесса:
	 + Наследование общих параметров:
		 + Окружение
		 + Права и группы
		 + Таблицы сигналов и открытых файлов
	 + Уникальность идентификаторов процессов
	 + Уникальность адресного пространства
		 + А для _нитей_ — легковесных процессов — нет
	 + Изменение параметров (перенаправление потоков ввода-вывода, например)

```console
$ { cat | grep hello; echo Success; } < file.txt > res.log
$ ( cat | grep hello; echo Success; ) < file.txt > res.log
$ ( cat | grep hello; echo Success; : ) < file.txt > res.log

```

 + exec — Задание программы процессу
	 + Получается _дерево процессов_ (`pstree`)
	 + Здесь меняется контекст процесса

```console
$ bash -c "exec cal" 
```

 + Отслеживание параметров процесса: 
	 + Управление (`ptrace`, `strace`)
	 + «Обмен сообщениями»
		 + Сигналы

```console
$ kill -TERM 12345
$ kill -INT 12345
$ kill -KILL 12345
$ kill -STOP 12345
$ kill -CONT 12345
$ kill -l
```

		 + IPC
		 + Реальный обмен сообщениями
+ Завершение
	+ Ожидание завершения
	+ Независимая работа процессов

## Простое управление процессами
 + `kill` и сигнальные команды
 + Управление фоновыми процессами — `bg`, `fg`
 + Связи между процессами — `jobs`, `disown`
